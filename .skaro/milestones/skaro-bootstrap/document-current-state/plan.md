# План

## Этап 1
Цель: изучить поверхность репозитория и извлечь реальную архитектуру.

Входы:
- `README.md`
- `pyproject.toml`
- `mdrj/*.py`
- `tests/*.py`

Выходы:
- базовое понимание компонентов, потоков и ограничений

Критерии готовности:
- определены основные модули, runtime flow и поведение, подтвержденное тестами

## Этап 2
Цель: материализовать совместимую со Skaro структуру проекта.

Входы:
- результаты stage 1
- документация Skaro по структуре директории и core concepts

Выходы:
- дерево `.skaro/` с основными проектными артефактами
- `.skaroignore`
- записи в корневом `.gitignore` для Skaro secrets/usage data

Критерии готовности:
- в репозитории есть минимальный устойчивый набор артефактов, ожидаемый Skaro для импортированного проекта

## Этап 3
Цель: оставить трассируемую точку передачи контекста для будущих LLM-сессий.

Входы:
- созданные артефакты

Выходы:
- task-local `AI_NOTES.md`
- указатель на Skaro-артефакты в README

Критерии готовности:
- следующий участник может быстро понять, где находится проектная истина и что нужно обновлять после изменений
