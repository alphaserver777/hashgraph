# Спецификация

## Задача
Представить текущее состояние репозитория в артефактах Skaro так, чтобы новая LLM-сессия могла понять систему, ограничения и следующие шаги без повторного reverse engineering всей кодовой базы.

## Объем
- Добавить корневую структуру `.skaro/`, необходимую для импортированного проекта.
- Написать базовые артефакты: конституцию, архитектуру, инварианты, обзор, эксплуатационные заметки и план разработки.
- Создать одну milestone/task-ветку как образец того, как дальше должен вестись проект.

## Вне объема
- Перепроектирование логики репликации или консенсуса.
- Настройка реального ключа LLM-провайдера.
- Изменение runtime-поведения, если только документация не требует небольшого видимого обновления репозитория.

## Критерии готовности
- `.skaro/` существует и пригодна для коммита.
- Артефакты точно описывают текущее состояние кода.
- Как минимум одна milestone/task-ветка содержит заметки реализации и верификацию.
