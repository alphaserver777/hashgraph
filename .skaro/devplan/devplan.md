# План разработки

## Цель
Превратить прототип MDRJ-DAG в репозиторий, в который любая LLM сможет безопасно входить повторно: быстро понимать текущее поведение, изменять его без нарушения инвариантов и оставлять после себя обновленные проектные артефакты.

## Этап 1: Bootstrap через Skaro
Статус: в работе

### Задача 1: задокументировать текущее состояние репозитория в артефактах Skaro
- Статус: выполнено
- Результат:
  - написана конституция
  - зафиксирован базовый срез архитектуры
  - записаны инварианты и обзор рисков
  - добавлены эксплуатационные заметки на уровне репозитория

### Задача 2: формализовать действующие protocol и storage contracts
- Статус: далее
- Фокус:
  - reference по схеме event/envelope
  - примеры request/response для endpoint
  - заметки по схеме хранения и стратегия миграций

### Задача 3: отделить production-границы от prototype-границ
- Статус: далее
- Фокус:
  - модель безопасности
  - ожидания по развертыванию
  - что относится только к demo, а что считается поддерживаемым ядром

## Этап 2: Укрепление архитектуры
