# Инварианты

> Архивный документ. Для текущей работы ориентируйтесь на документы в [`docs/`](/home/admsys/hashgraph/docs). Если потребуется снова формализовать инварианты как отдельный рабочий документ, это нужно сделать внутри `docs/`, а не здесь.

## Инварианты графа
- Сохраненный граф должен оставаться ацикличным. [`DAGStorage.toposort()`](/home/admsys/hashgraph/mdrj/storage.py) выбрасывает ошибку при цикле; ни одна функция не должна обходить это предположение.
- Каждое не-genesis событие по возможности должно ссылаться максимум на двух родителей и стараться иметь не меньше двух ссылок, если доступны self-parent, удалённый кандидат и/или anchors.
- Ребра parent-child выводятся из списка родителей события и должны оставаться синхронизированными с фактом сохранения события.

## Инварианты репликации
- Идентификаторы событий неизменяемы и глобально дедуплицируются первичным ключом.
- Повторный ingest уже известного события может merge-ить `path_meta`, но не должен создавать дубликаты строк события или менять identity payload.
- Gossip обязан ставить вновь сохраненные события в очередь на дальнейшую репликацию.

## Инварианты упорядочивания
- `consensus_ts` должен вычисляться детерминированно из локально наблюдаемого состояния события и envelope, после чего итоговый порядок разрешается через tie-break по `event.id`.
- Если меняется семантика порядка, интеграционные тесты должны подтверждать сходимость между узлами после снятия partition.

## Инварианты хранения
- SQLite является источником истины для событий, ребер, envelope и сохраненного состояния пиров.
- В пустом хранилище genesis anchors должны появиться до начала обычной эмиссии событий.
- Запрос frontier должен отражать листовые вершины в соответствии с таблицей `edges`.

## Инварианты интерфейсов
- CLI-команды должны оставаться тонкими обертками над HTTP endpoint или helper-логикой жизненного цикла узла.
- API payload должны оставаться canonical JSON object/array, как задокументировано в [`README.md`](/home/admsys/hashgraph/README.md).
- Конфиги должны оставаться YAML-ориентированными и корректно маппиться на [`NodeConfig`](/home/admsys/hashgraph/mdrj/config.py).

## Инварианты документации
- Актуальные изменения формы события, формы endpoint, жизненного цикла узла или сценария развертывания должны отражаться в `docs/`.
- Stage-структура и `AI_NOTES.md` внутри `.skaro/` больше не являются обязательной частью текущего процесса.
