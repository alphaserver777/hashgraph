# Инварианты

## Инварианты графа
- Сохраненный граф должен оставаться ацикличным. [`DAGStorage.toposort()`](/home/admsys/work/hashgraph/mdrj/storage.py) выбрасывает ошибку при цикле; ни одна функция не должна обходить это предположение.
- Каждое не-genesis событие по возможности должно ссылаться как минимум на двух родителей, если это позволяют anchors и frontier.
- Ребра parent-child выводятся из списка родителей события и должны оставаться синхронизированными с фактом сохранения события.

## Инварианты репликации
- Идентификаторы событий неизменяемы и глобально дедуплицируются первичным ключом.
- Повторный ingest уже известного события может merge-ить `path_meta`, но не должен создавать дубликаты строк события или менять identity payload.
- Gossip обязан ставить вновь сохраненные события в очередь на дальнейшую репликацию.

## Инварианты упорядочивания
- Consensus timestamp должен вычисляться детерминированно из наблюдаемых сэмплов, после чего итоговый порядок разрешается через tie-break по `event.id`.
- Если меняется семантика порядка, интеграционные тесты должны подтверждать сходимость между узлами после снятия partition.

## Инварианты хранения
- SQLite является источником истины для событий, ребер, envelope и сохраненного состояния пиров.
- В пустом хранилище genesis anchors должны появиться до начала обычной эмиссии событий.
- Запрос frontier должен отражать листовые вершины в соответствии с таблицей `edges`.

## Инварианты интерфейсов
- CLI-команды должны оставаться тонкими обертками над HTTP endpoint или helper-логикой жизненного цикла узла.
- API payload должны оставаться canonical JSON object/array, как задокументировано в [`README.md`](/home/admsys/work/hashgraph/README.md).
- Конфиги должны оставаться YAML-ориентированными и корректно маппиться на [`NodeConfig`](/home/admsys/work/hashgraph/mdrj/config.py).

## Инварианты документации
- Любое изменение формы события, формы endpoint, жизненного цикла узла или сценария развертывания должно обновлять `.skaro/architecture/architecture.md` и этот файл.
- Любая реализация stage должна оставлять проверяемый след в `.skaro/milestones/.../stages/.../AI_NOTES.md`.
