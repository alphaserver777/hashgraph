# Корпус научного материала к диссертации

Папка содержит рабочие документы для защиты решения MDRJ-DAG —
распределённого журнала событий ИБ на базе hashgraph. Каждый файл
самостоятельный, читается отдельно; перекрёстные ссылки даны
явно в тексте.

## Состав

**Обзорные документы (широкий мазок, для научного руководителя и оппонентов)**

- [`analysis-and-novelty.md`](analysis-and-novelty.md) — системный
  анализ слабых сторон и научной новизны (7 групп ограничений,
  4 пункта научной новизны, 4 практической, 7 направлений развития).
- [`memory-profile.md`](memory-profile.md) — почему K_рес у A4
  нелинеен, диагноз семи точек роста памяти, три архитектурных
  слоя сдерживания, эмпирическая верификация.
- [`metric-mapping.md`](metric-mapping.md) — мост между runtime-
  метриками Prometheus и переменными стохастической модели
  (`simulation.html`), PromQL для калибровки A4.

**Углублённые исследовательские заметки (узкие вопросы по запросам комиссии)**

- [`event-selection.md`](event-selection.md) — какие события и
  почему попадают в реестр; четыре класса критериев включения,
  обоснование классификации A/B/C через профиль ответа, шумовые
  события.
- [`replication-tradeoff.md`](replication-tradeoff.md) —
  нужна ли полная репликация принципиально; альтернативные модели
  (witness-only, эрезурное кодирование, hot/cold по классу,
  retention skeleton), сравнительная таблица overhead,
  рекомендации для текущей работы.

## Заделы для будущих заметок

Добавляются по мере возникновения вопросов на защите / у научрука.
Список — не план, а перечень тем, готовых к раскрытию:

- *Формальная верификация checkpoint-протокола* — TLA+ /
  mCRL2 спецификация свойств безопасности (нет двух confirmed
  с разным merkle_root) и живости (новый confirmed появляется
  при ≥ 2/3 честных).
- *Цепи Маркова для УБИ.124* — стохастическая модель атаки на
  $M \le \lfloor N/3 \rfloor$ узлов, замкнутая формула
  $P_{\text{undetected}}(N, M, T_{\text{checkpoint}})$.
- *Сравнительный анализ* — qualitatively/quantitatively с
  Hyperledger Fabric (CFT), IPFS-pinned журналами (eventual
  consistency без BFT), TPM-attestation решениями (хост-уровень).
- *Углублённый маппинг на ФСТЭК* — ОУД, СТО БР ИББС, типовые
  требования аттестации СЗИ. Сейчас каждое event_kind помечено
  через `linked_threats` — нужно собрать обратное соответствие.
- *Cross-node verify* — почему локальный verify недостаточен
  (узел может лгать о результате) и как организовать
  взаимопроверку.
- *Привязка времени* — RFC 3161 TSA для долгого хранения улик
  как доказательства. Текущий ts_local подвержен дрифту часов.

## Принцип формата

Каждая заметка — научное рассуждение с явной структурой:

1. **Постановка задачи** (что формализуется, какое свойство
   требуется обеспечить);
2. **Разбор альтернатив** (что было предложено в литературе /
   практике);
3. **Аргументированная рекомендация** (что выбрано для текущей
   работы и почему);
4. **Открытые вопросы** (что осталось для следующей итерации).

Не план реализации, не отчёт о работе, не справочник по коду.
Имплементационные подробности живут в `mdrj/`, в `data/event_catalog.json`
и в коммитах git.
